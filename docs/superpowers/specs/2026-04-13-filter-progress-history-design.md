# Design: Filter, No-Repeat Draw & Persistent History

**Date:** 2026-04-13
**Status:** Approved

## Overview

Enhance the Taiwan Railway Station lottery app with three interconnected features:
1. Quick tag-based filtering for routes and regions
2. No-repeat draw mode with "conquer all" progress tracking
3. Persistent history with timestamps via localStorage

All features are implemented in the single-file HTML app generated by `generate_app.py`.

## 1. Quick Tag Filter

### UI

A row of clickable tag buttons placed **above** the draw count selector, inside the `.controls` section. Two groups displayed in two rows:

**Route tags:** `縱貫線` `海岸線` `山線` `宜蘭線` `北迴線` `南迴線` `臺東線` `屏東線` `支線`
- "支線" combines: 平溪線, 內灣線, 集集線, 深澳線, 沙崙線, 六家線, 花蓮臨港線

**Region tags:** `北部` `中部` `南部` `東部`
- 北部 = 基隆市, 新北市, 臺北市, 桃園市, 新竹縣, 新竹市
- 中部 = 苗栗縣, 臺中市, 彰化縣, 南投縣, 雲林縣
- 南部 = 嘉義縣, 嘉義市, 臺南市, 高雄市, 屏東縣
- 東部 = 宜蘭縣, 花蓮縣, 臺東縣

### Behavior

- All tags are **selected by default** (highlighted style)
- Clicking a tag toggles it on/off
- The stats bar updates in real-time: `候選 N 站` reflects the current filtered count
- The candidate pool for drawing is recomputed on every tag change
- Filter state is NOT persisted (resets on page reload) — keeps it simple
- If all tags are deselected, show a warning and disable the draw button

### CSS

- Active tag: gradient background matching app theme (`#6366f1` to `#818cf8`), white text
- Inactive tag: dark background (`#1e293b`), muted text (`#64748b`)
- Tags wrap naturally across lines, small gap between them
- On mobile: smaller font and padding

## 2. No-Repeat Draw + Conquer-All Progress

### No-Repeat Logic

- Maintain a Set `drawnStations` in JS, persisted to `localStorage` key `train-random-drawn`
- Before each draw, filter out stations already in `drawnStations` from the current candidate pool
- After drawing, add winners to `drawnStations` and save
- If no stations remain in the filtered pool, show a message: "所有候選站都已抽過！" and disable draw button

### Progress Display

- Add a progress indicator in the stats bar: `制霸 12/138`
- Below the stats, add a thin progress bar (full width of panel)
  - Gradient fill matching app theme
  - Shows `drawnStations.size / total candidates count`
- The denominator (138) is fixed — it counts all non-express stations regardless of current filter
- The progress bar and counter always reflect total progress, not filtered progress

### Conquer-All Celebration

- When `drawnStations.size === total candidates count` (138), trigger a special celebration:
  - Extended confetti animation (longer, more particles)
  - A congratulatory overlay message: "恭喜全制霸！" with a trophy emoji
  - The overlay auto-dismisses after 4 seconds

### Reset

- A small "重置" (reset) text button placed next to the progress counter
- Clicking shows a confirm dialog: "確定要重置制霸進度嗎？所有抽籤記錄將被清除"
- On confirm: clear `drawnStations`, clear history, update UI, save to localStorage

## 3. Persistent History with Timestamps

### Data Structure

```javascript
// localStorage key: "train-random-history"
// Value: JSON array of objects
[
  { "name": "三坑", "city": "基隆市", "line": "縱貫線", "time": "2026-04-13T14:30:00" },
  ...
]
```

### UI Changes

- Each history tag shows: `4/13 三坑` (month/day + station name)
- History section is always visible (remove `display:none` default)
- Show "尚無記錄" placeholder when empty
- Cap at 50 most recent entries in display; store all in localStorage
- History loads from localStorage on page init

### Drawn Set

```javascript
// localStorage key: "train-random-drawn"
// Value: JSON array of station names
["三坑", "暖暖", "猴硐", ...]
```

Loaded on init into the `drawnStations` Set.

## Implementation Scope

### Files Modified

- `generate_app.py` — all changes are in this single file:
  - New CSS for filter tags, progress bar
  - New HTML for filter row, progress bar
  - New JS for filter logic, localStorage read/write, drawn tracking, conquer celebration

### Generated Output

- `draw_station.html` — regenerated
- `index.html` — copied from draw_station.html

### What's NOT Included

- No server-side storage — purely client-side localStorage
- No filter state persistence — resets on reload
- No "visited" checkbox — drawn automatically counts as progress
- No action links (Google Maps, timetable) — deferred to future iteration
- No share functionality — deferred to future iteration
